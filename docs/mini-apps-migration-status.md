design-system canonicalization (the "website-roast treatment"). The app's
`status` in `src/app/mini-apps/_data/apps.ts` mirrors this:

- **`live`** = reviewed, approved, done.
- **`draft`** = not yet reviewed. App still works/launches; the marketplace card
  shows an amber DRAFT badge so done-vs-not is visible at a glance.

audit (idle / loading / result / error states, spacing, export PNG+PDF) and you
approve it. Reference checklist: `docs/mini-apps-canonical-checklist.md`.

## Shared loading abstraction (DRY)

The per-app loading state machine (rAF progress + staged timers + the 2-col
stage-card UI) was duplicated across all 16 live apps. It is now **one shared
definition** used by every live app:

- `src/components/mini-apps/useMiniAppLoader.ts` — the state machine hook
  (`start/stop/complete/reset` + `progressPct/loadingPct/latency/activeStage/
doneStages/stageLogs/waiting`).
- `src/components/mini-apps/LoadingStages.tsx` — the progress bar + stage-card UI.
- `src/styles/mini-app-ui.css` §3b — the canonical `.progress-track`/`.stages`/
  `.stage`/… (per-app copies removed).

Two bugs fixed once in the hook: (1) latency froze at the 98% cap because the
rAF loop stopped — it now keeps ticking so `LAT` reflects the real wait;
(2) the bar looked stuck at 98% — `.progress-bar.waiting` now pulses/dims.
Verified: build + typecheck + lint clean across all 16; loading renders via the
shared component (incl. the `tsf-`-prefixed tech-stack-finder outlier).
