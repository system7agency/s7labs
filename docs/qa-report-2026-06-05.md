## Cross-cutting issues

### CC-1 — Form error styling broken on every mini-app (prettier strips space in conditional className)

**Severity:** High
**Found on:** pricing-diagnostic (Section 3 / check 2 — empty submit)
**Affects:** 13 mini-app pages — agentic-readiness, ai-overview-tracker, ai-visibility-score, automation-blueprint, crm-sanity, job-brief, linkedin-hook, outbound-radar, pricing-diagnostic, proposal-engine, share-of-voice, tech-stack-recommender, website-roast.

**Symptom:** Submitting an empty form (or invalid input) collapses the input layout. The submit-arrow button disappears, the error message and example pills overlap. See screenshot in QA conversation.

**Root cause:** Same prettier-tailwindcss bug as the earlier "stage" className regression — `className={\`<base>${flag ? 'error' : ''}\`}`gets its leading space stripped by the formatter at commit time, rendering`<base>error`(one fused word) instead of`<base> error` (two classes). The base CSS rule never matches, so layout collapses.

**Fix recipe:** Swap each occurrence to `clsx('<base>', { error: <flag> })`. ~19 edits across the 13 files. Same recipe used to fix the stage className regression earlier.

**Status:** ✅ Fixed mid-QA in commit `89bceef` on this branch. 19 occurrences swapped to `clsx(...)` across 13 mini-apps. Awaiting re-test on pricing-diagnostic Section 3 below.

### CC-2 — Email gate runs AFTER the model call (credit waste on dropoffs)

**Severity:** High
**Found on:** pricing-diagnostic (Section 4 review)
**Affects:** All 11 live mini-apps (the original `<EmailGate>` overlay pattern).

**Symptom:** A visitor types a URL, clicks submit, the Anthropic call fires immediately, then the email gate appears over the result. If they bail at the gate (or enter a disposable / free-provider email), we have already paid for the model run and lost the lead. Free-provider emails (gmail / yahoo / outlook / etc.) were also accepted because the server only checked disposable domains.

**Fix recipe:**

1. Add a `FREE_EMAIL_PROVIDERS` set + `isFreeEmailProvider()` to `@/lib/leads/disposable` and reject in `/api/leads/submit`. (Shared, lands once.)
2. Per mini-app: add an inline email field to the main form alongside the URL/input. On submit, validate URL + email client-side, then `POST /api/leads/submit` (which now also enforces disposable + free-provider). Only fire the Anthropic call on success. After the model returns, `POST /api/leads/complete` with the submission id + output + cost.
3. Remove the `<EmailGate>` + `<SubmitOnce>` wrappers around the result panel (no longer needed).
4. Also relabel stale `pending` submissions (older than 5 minutes) as `ABANDONED` in the Insights dashboard so orphan rows don't read as "in progress" forever.

**Status:** ✅ Rolled out to **pricing-diagnostic** on this branch. Remaining 10 live apps still need the same pattern applied: `outbound-radar`, `job-brief`, `linkedin-hook`, `crm-sanity`, `website-roast`, `email-finder`, `find-people`, `agentic-readiness`, `ai-overview-tracker`, `ai-visibility-score`. Will apply during their respective sections of this QA pass.

### CC-3 — Error section silently invisible on every mini-app (CC-1 cousin)

**Severity:** High
**Found on:** pricing-diagnostic Section 7 check 2 (unreachable URL → empty panel below the sysstate readout, see screenshot in QA conversation).
**Affects:** All 11 live mini-apps. Same prettier-tailwindcss space-strip root cause as CC-1, but on the per-state visibility toggle (`pd-state`, `wr-state`, `jb-state`, etc.) instead of the form error class.

**Symptom:** Hitting an error path renders an empty panel. The sysstate readout flips to `SYS ERROR` but the error section's content (icon, message, "Try again" button) never appears.

**Root cause:** Each mini-app has four section toggles like `className={\`<prefix>-state${flag ? 'active' : ''}\`}` — prettier strips the leading space inside the ternary, fusing the modifier with the prefix. For idle/loading/result this *accidentally* works because the fused class (`<prefix>-stateactive`) doesn't match `.<prefix>-state { display: none }`, so the section falls through to default `display: block`. But the error section's template is `\`<prefix>-state error-state${...}\``— the leading`<prefix>-state `(with a space) always matches the`display: none` rule, so the section stays hidden even when active.

**Fix recipe:** Same as CC-1 — swap each to `clsx('<prefix>-state', { active: <flag> })`. 4 occurrences per file × 11 files = 44 edits.

**Status:** ✅ Fixed on **pricing-diagnostic** this branch. Remaining 10 apps to fix during their QA sections (or as a single sweep — flagged for decision).
